# 算法复杂度分析 — 学习笔记

## 一、时间复杂度

时间复杂度描述算法执行时间随输入规模增长的变化趋势，使用大O表示法。

### 常见时间复杂度（从快到慢）

- **O(1) — 常数时间**：不随输入增长，如数组索引访问、哈希表查找
- **O(log n) — 对数时间**：每次操作将问题规模减半，如二分查找
- **O(n) — 线性时间**：与输入规模成正比，如遍历数组
- **O(n log n) — 线性对数时间**：如归并排序、快速排序（平均情况）
- **O(n²) — 平方时间**：如冒泡排序、选择排序
- **O(2ⁿ) — 指数时间**：如递归求解斐波那契数列（无记忆化）

### 分析方法

1. 找到执行次数最多的基本操作
2. 用输入规模 n 表示该操作的执行次数
3. 忽略常数系数和低阶项
4. 保留最高阶项作为大O表示

## 二、空间复杂度

空间复杂度描述算法执行所需的额外存储空间随输入规模的增长趋势。

重要概念：
- **原地算法（In-place）**：额外空间为 O(1)，直接在输入数据上操作
- **递归调用栈**：递归算法的空间复杂度要考虑调用栈深度

## 三、哈希表（Hash Table）

哈希表是实现字典和集合的底层数据结构。

核心操作：
- **哈希函数**：将任意大小的数据映射到固定大小的索引
- **冲突解决**：当两个不同的键映射到同一索引时的处理策略
  - 链地址法（Separate Chaining）：每个桶是一个链表
  - 开放寻址法（Open Addressing）：寻找下一个空位

Python字典的实现：
- 使用开放寻址法
- 负载因子超过2/3时自动扩容
- 扩容时重新哈希所有元素（rehashing）

## 四、排序算法对比

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | 是 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 否 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | 是 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 是 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | 否 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 否 |
| Timsort | O(n) | O(n log n) | O(n log n) | O(n) | 是 |

## 五、递归与迭代

递归：函数调用自身来解决问题。
- 必须要有基准条件（base case）防止无限递归
- 每次递归调用产生新的栈帧，消耗内存
- 代码简洁但可能有性能代价

迭代：使用循环（for/while）重复执行代码块。
- 通常比递归效率高（无函数调用开销）
- 所有递归都可以转化为迭代（使用显式栈）
- 某些问题（树的遍历）用递归更自然
